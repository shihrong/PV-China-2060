# Code and Data for Solar PV Power Potential Assessment with WRF-Chem

This repository contains the source code, numerical model configurations, and core datasets required to reproduce the findings and figures presented in our manuscript. The pipeline links regional atmospheric chemistry-climate modeling (**WRF-Chem**) with an advanced solar engineering physics engine (**pvlib-python**) to evaluate future solar photovoltaic (PV) power potential ($P_{dc}$).

## 2. Core Modules & Usage Instructions

### Part I: WRF-Chem Numerical Configuration (`0_*`)

The `0_` prefix files contain standard runtime controls for running the Weather Research and Forecasting model coupled with Chemistry (**WRF-Chem**).

* **Namelist Tuning:** `0_namelist.input_SV2` and `0_namelist.input_SV2.cheminit` define critical boundary layer parameters, microphysics configurations, aerosol emission mechanisms, and simulation time steps. To use them, copy these files to your compiled WRF run directory and rename the target file to `namelist.input`.
* **Job Execution:** `0_batch_real_bc_wrf_1.csh` is configured for high-performance computing (HPC) environments running C-Shell. It automates sequential system initialization (`real.exe`) and parallel MPI simulation binding (`wrf.exe`). Execute it via your cluster scheduler (e.g., `qsub` or `sbatch`).

### Part II: PV Yield Mathematical Optimization Engine (`1_*`)

The `1_PV_Potential_Power_func.py` script houses the core physical pipeline translating atmospheric parameter arrays generated by WRF-Chem directly into continuous spatio-temporal PV yield vectors.

#### Physical Modeling Pipeline Architecture:

```
[WRF-Chem Input Matrix] (SWDOWN, T2, U10/V10 Wind Components, HGT)
           │
           ▼
[Solar Position Tracking] ──► NREL SPA Engine (Precise Zenith/Azimuth calculations)
           │
           ▼
[Irradiance Decomposition] ──► DISC Model (Decomposes GHI to DNI & DHI arrays)
           │
           ▼
[Plane-of-Array Transposition] ──► Perez Model (Fixed south-facing tilt matched to local latitude)
           │
           ▼
[Optical & Spectral Losses] ──► Fresnel + Xie Reflection and Sandia Air Mass (SAPM) Coefficients
           │
           ▼
[Dynamic Thermal Modeling] ──► SAPM Cell Model (Translates ambient T2 & Wind Speed into T_cell)
           │
           ▼
[Electrical DC Power Yield] ──► PVWatts Engine (Simulates active output for a standard 1kW array)

```

### Part III: Analytical Visualization & Figure Reproduction (`8_*`)

The `8_` series Python scripts extract processed matrix records from the provided `.hdf5` and `.xlsx` target summary data collections to map exact figures embedded inside the primary text and supplementary material.

* **Prerequisites:** Ensure all supporting `.hdf5` and `.xlsx` matrices sit inside the identical directory tier as the plotting scripts.
* **Execution:** Run individual scripts via terminal terminal inputs to render the graphical plots, for example:
```bash
python 8_Fig1_map_diff.py
python 8_FigS2_violin.py

```
